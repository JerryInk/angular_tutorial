## Неделя 3. Новый синтаксис управления шаблоном (Control Flow)
------------------------------
## ЛЕКЦИЯ 3: Декларативный синтаксис шаблонов в Angular 22
Продолжительность: 2 академических часа (90 минут)
## 1. Эволюция шаблонизатора: От структурных директив к нативному Control Flow (20 минут)
До версии Angular 17 управление логикой в HTML-шаблонах осуществлялось с помощью структурных директив *ngIf, *ngFor и *ngSwitch. Этот подход имел ряд серьезных архитектурных и эргономических недостатков:

* Синтаксический оверхед: Директивы требовали импорта специального модуля CommonModule, что противоречило идее легковесных Standalone-компонентов.
* Искусственные HTML-теги: Для группировки элементов без изменения структуры DOM приходилось использовать служебный тег <ng-container>, что усложняло чтение разметки.
* Слабая интеграция с типами: Старый синтаксис плохо поддерживал строгую типизацию TypeScript внутри шаблона.

Начиная с Angular 17 и получив финальное развитие в Angular 22, фреймворк перешел на встроенный язык управления потоком (Control Flow), использующий синтаксис с символом @. Он работает на уровне компилятора, не требует никаких импортов, обеспечивает идеальную проверку типов и выполняется в разы быстрее.
## 2. Ветвление с помощью @if, @else if и @else (20 минут)
Новая конструкция выглядит как стандартный JavaScript/TypeScript, перенесенный прямо в HTML-разметку:
```html
  @if (user.role === 'admin') {
    <button>Панель администратора</button>
  } @else if (user.role === 'student') {
    <button>Моё обучение</button>
  } @else {
    <p>Доступ ограничен</p>
  }
```
Важная фича (Локальные переменные): Мы можем сохранить результат асинхронного вычисления или Сигнала в локальную переменную шаблона с помощью ключевого слова as: @if (currentCourse(); as course) { ... }. Это избавляет от необходимости многократно вызывать Сигнал в коде.
## 3. Высокопроизводительные циклы: @for и магия параметра track (30 минут)
Вывод списков данных теперь реализуется через блок @for. Его синтаксис требует обязательного указания выражения track:
```html
  @for (task of tasks(); track task.id) {
    <div class="task-item">{{ task.title }}</div>
  }
```
## Почему параметр track обязателен?
В старом *ngFor указание функции trackBy было опциональным. Если разработчик забывал её написать, Angular при любом изменении массива (например, добавлении одного элемента в конец) полностью уничтожал все DOM-узлы списка и перерисовывал их заново.
В Angular 22 компилятор жестко требует track. Вы обязаны указать уникальный идентификатор элемента (например, track item.id или track item.uid). Благодаря этому алгоритм рендеринга точно знает, какой именно элемент массива изменился, переместился или удалился. DOM-узлы остальных элементов не затрагиваются. Это обеспечивает максимальную производительность в Zoneless/OnPush режиме.
## Бонусные конструкции @for:

* Блок @empty: Срабатывает автоматически, если переданный массив пуст. Больше не нужно писать отдельный @if (tasks.length === 0) вокруг цикла.
* Контекстные переменные: Внутри цикла доступны встроенные переменные: $index (текущий индекс), $first (флаг первого элемента), $last (флаг последнего элемента), $even и $odd (четный/нечетный).

## 4. Группировка условий в @switch (20 минут)
Блок @switch осуществляет эквивалентное JS ветвление по значению.
Эксклюзивная фича Angular 22: Поддержка множественных условий (мульти-кейс) через точку с запятой.
Если для статусов 'loading' и 'processing' нужно отрендерить один и тот же лоадер, теперь их можно сгруппировать: @case ('loading'; 'processing') { <app-spinner/> }.

## ПРАКТИЧЕСКОЕ ЗАНЯТИЕ 3: Вывод динамического каталога курсов из базового бэкенда
Продолжительность: 2 академических часа (90 минут)
Цель работы: Научиться профессионально использовать блоки @if, @for, @empty и @switch в Angular 22, освоить работу с контекстными переменными цикла ($index, $first) и реализовать динамическую фильтрацию списка элементов в OnPush-компоненте.
## Шаг 1: Подготовка компонента каталога (15 минут)

   1. Сгенерировать компонент списка курсов / задач:
   
   ng g c components/course-list
   
   2. Открыть course-list.component.ts, импортировать ChangeDetectionStrategy и активировать режим OnPush:
   
   import { Component, OnInit, ChangeDetectionStrategy, signal } from '@angular/core';
   interface Course {
     id: number;
     title: string;
     status: 'active' | 'completed' | 'archived';
     hours: number;
   }
   
   @Component({
     selector: 'app-course-list',
     templateUrl: './course-list.component.html',
     styleUrl: './course-list.component.scss',
     changeDetection: ChangeDetectionStrategy.OnPush
   })export class CourseListComponent implements OnInit {
     // Реактивный Сигнал для хранения массива данных
     courses = signal<Course[]>([]);
   
     // Сигнал для текущего фильтра статуса
     currentFilter = signal<string>('all');
   
     ngOnInit(): void {
       // Имитируем получение базовой части данных с REST API
       this.courses.set([
         { id: 101, title: 'Разработка веб-приложений на Angular 22', status: 'active', hours: 32 },
         { id: 102, title: 'Основы баз данных (PostgreSQL)', status: 'completed', hours: 64 },
         { id: 103, title: 'Проектирование систем (Запросы и Архитектура)', status: 'active', hours: 16 },
         { id: 104, title: 'Администрирование Linux (Архивный курс)', status: 'archived', hours: 48 }
       ]);
     }
   }
   
   
## Шаг 2: Реализация логики фильтрации (15 минут)
Поскольку мы работаем в OnPush-компоненте, напишем метод, который будет менять значение фильтра, и современную стрелочную функцию (фича шаблонов Angular 22) для фильтрации элементов прямо «на лету».

  // Внутри класса CourseListComponent
  setFilter(filterValue: string): void {
    this.currentFilter.set(filterValue);
    // Сигнал обновился -> Начнется точечный рендеринг шаблона
  }

  // Вспомогательный метод-предикат, который мы вызовем в HTML
  isMatch(courseStatus: string): boolean {
    if (this.currentFilter() === 'all') return true;
    return courseStatus === this.currentFilter();
  }

## Шаг 3: Написание шаблона с использованием нового Control Flow (35 минут)

   1. Открыть course-list.component.html и реализовать интерфейс управления и вывода:
   
   <div class="catalog-container">
     <h3>Каталог учебных курсов (Семестр 5)</h3>
   
     <!-- Кнопки управления фильтрацией (Event Binding) -->
     <div class="filter-actions">
       <button (click)="setFilter('all')" [class.active]="currentFilter() === 'all'">Все</button>
       <button (click)="setFilter('active')" [class.active]="currentFilter() === 'active'">Активные</button>
       <button (click)="setFilter('completed')" [class.active]="currentFilter() === 'completed'">Завершенные</button>
     </div>
   
     <div class="courses-grid">
       <!-- Цикл @for с обязательным параметром track. Применяем стрелочный предикат v22 -->
       @for (course of courses(); track course.id; let idx = $index; let isFirst = $first) {
   
         <!-- Выводим только те элементы, которые подходят под фильтр -->
         @if (isMatch(course.status)) {
           <div class="course-card" [class.featured-card]="isFirst">
   
             <span class="index-badge">#{{ idx + 1 }}</span>
             <h4>{{ course.title }}</h4>
             <p>Объем: {{ course.hours }} акад. часов</p>
   
             <!-- Демонстрация синтаксиса @switch -->
             @switch (course.status) {
               @case ('active') {
                 <span class="status green">В процессе изучения</span>
               }
               @case ('completed') {
                 <span class="status blue">Курс пройден</span>
               }
               @default {
                 <span class="status gray">В архиве</span>
               }
             }
           </div>
         }
   
       } @empty {
         <!-- Автоматический блок на случай отсутствия курсов в массиве -->
         <div class="empty-state">
           <p>Список курсов пуст или данные не загружены с бэкенда.</p>
         </div>
       }
     </div>
   </div>
