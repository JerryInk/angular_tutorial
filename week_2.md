## Неделя 2. Компоненты, привязка данных и жизненный цикл в OnPush
------------------------------
## ЛЕКЦИЯ 2: Связывание данных и хуки жизненного цикла в Zoneless-архитектуре
Продолжительность: 2 академических часа (90 минут)

## 1. Механизмы рендеринга и триггеры обновления UI (20 минут)
В Angular 22 при использовании ```ChangeDetectionStrategy.OnPush``` и отсутствии Zone.js шаблон компонента не перерисовывается сам по себе при изменении обычных переменных класса. Чтобы интерфейс обновился, Angular должен получить явный сигнал.
Рендеринг в OnPush-компоненте триггерится тремя событиями:

* Изменение значения Angular Signal, который напрямую вызван в HTML-шаблоне.
* Изменение ссылки на входящие данные через механизм ```input``` (получение новых данных от родителя).
* Срабатывание обработчика событий (Event Binding) внутри самого компонента (например, клик по кнопке).

## 2. Виды привязки данных в шаблонах (25 минут)
Для построения интерактивного интерфейса Angular использует три основных типа привязки:

   1. Интерполяция ```{{ expression }}```: Вывод текстовых значений в HTML. Внутри фигурных скобок можно писать простые JS-выражения, тернарные операторы и — начиная с Angular 22 — вызывать стрелочные функции.
   2. Привязка свойств ```[property]="value"```: Передача данных из TypeScript-класса в свойства DOM-элементов или ```@Input``` свойства дочерних компонентов (например, ```[disabled]="isFormInvalid"``` или ```[src]="imageUrl"```).
   3. Привязка событий ```(event)="handler()"```: Перехват действий пользователя (клик, ввод текста, отправка формы) и вызов методов класса (например, ```(click)="saveTask()"```).

## 3. Жизненный цикл компонента в мире без Zone.js (25 минут)
Каждый компонент Angular проходит строгую последовательность этапов от создания до уничтожения. В Zoneless/OnPush режиме понимание хуков жизненного цикла критично, так как они определяют, когда данные готовы к отображению.
Основные хуки (вызываются как методы класса при импорте соответствующих интерфейсов):

* ```constructor()```: Стандартный конструктор класса. На этом этапе компонент еще не инициализирован Angular, свойства шаблона недоступны. Сам Angular ещё не успел связать класс с его HTML-шаблоном и еще не обработал входящие данные (inputs) от родительского компонента. Используется только для базового объявления переменных и инъекций зависимостей через inject().
* ```ngOnChanges()```: Вызывается до ```ngOnInit``` и каждый раз, когда изменяются входящие свойства (```[input]```), переданные от родительского компонента. Примечание: для новых сигнальных инпутов ```input()``` вместо этого хука эффективнее использовать ```computed()``` или ```effect()```.
* ```ngOnInit()```: Главная точка входа. Вызывается один раз после того, как Angular настроил все входные свойства компонента. Именно здесь выполняются первичные запросы к базовому бэкенду или инициализируются стартовые данные.
* ```ngOnDestroy()```: Вызывается непосредственно перед удалением компонента из DOM (например, при смене страницы в роутере). Используется для очистки памяти, отписки от асинхронных потоков и уничтожения таймеров во избежание утечек памяти.

## 4. Безопасная инициализация данных (20 минут)
Никогда не делать тяжелых вычислений и сетевых запросов в ```constructor()```. Конструктор должен оставаться «легким». Любая логика подготовки данных, работа с DOM или обращение к API бэкенда должны строго инкапсулироваться внутри метода ```ngOnInit()```.

## ПРАКТИЧЕСКОЕ ЗАНЯТИЕ 2: Разработка интерактивного UI-компонента профиля студента
Продолжительность: 2 академических часа (90 минут)
Цель работы: Освоить все виды привязки данных в шаблонах Angular 22, научиться использовать хуки ```ngOnInit``` и ```ngOnDestroy```, а также на практике увидеть разницу между обновлением обычных переменных и реактивных триггеров в режиме OnPush.

## Шаг 1: Создание компонента карточки профиля (15 минут)

   1. В терминале проекта сгенерировать новый компонент:
```bash
  ng g c components/student-profile
```
   2. Открыть student-profile.component.ts и обязательно зафиксировать стратегию OnPush:
```typescript
  import { Component, OnInit, OnDestroy, ChangeDetectionStrategy, signal } from '@angular/core';
   
  @Component({
    selector: 'app-student-profile',
    imports: [],
    templateUrl: './student-profile.component.html',
    styleUrl: './student-profile.component.scss',
    changeDetection: ChangeDetectionStrategy.OnPush // Включаем OnPush
  })
  export class StudentProfileComponent implements OnInit, OnDestroy {
    // Здесь будет логика
    ngOnInit(): void {}
    ngOnDestroy(): void {}
  }
```
   
## Шаг 2: Моделирование данных и работа с хуками (20 минут)

   1. Опишем внутреннее состояние компонента. Добавим обычную переменную (для демонстрации проблемы OnPush) и Сигнал (для правильного решения):
```typescript
  // Внутри класса StudentProfileComponent
  // 1. Обычная переменная (НЕ триггерит OnPush сама по себе)
  clickCount = 0;
  // 2. Реактивный Сигнал для данных профиля
  studentData = signal<{ name: string; group: string; avatar: string } | null>(null);
  // Имитируем таймер для демонстрации ngOnDestroy
  private timerId: any;
   
  ngOnInit(): void {
    console.log('--- Хук ngOnInit сработал ---');
    // Имитируем быструю загрузку базовой части данных с бэкенда через 1 секунду
    setTimeout(() => {
      this.studentData.set({
        name: 'Алексей Иванов',
        group: 'ИТ-302',
        avatar: 'https://dicebear.com'
      });
    }, 1000);
   
    // Запускаем фоновый интервал (метрика времени на странице)
    this.timerId = setInterval(() => {
      console.log('Студент просматривает профиль...');
    }, 2000);
  }
   
  ngOnDestroy(): void {
    console.log('--- Хук ngOnDestroy сработал. Очистка ресурсов ---');
    // Обязательно очищаем интервал, чтобы не было утечки памяти в SPA
    clearInterval(this.timerId);
  }

  // Метод для обработки клика
  badIncrement(): void {
    this.clickCount++; 
    console.log('Внутреннее значение clickCount:', this.clickCount);
    // Увидим: в консоли значение растет, но на экране цифра НЕ меняется!
  }
```
   
## Шаг 3: Верстка шаблона с привязкой данных (25 минут)

   1. Открыть student-profile.component.html и применить механизмы интерполяции, привязки свойств и событий:
```html
  <div class="profile-container">
    <h3>Личный кабинет студента</h3>
   
    <!-- Защита от null данных до того, как сработает setTimeout -->
    @if (studentData(); as student) {
      <div class="profile-card">
        <!-- Property Binding [src] -->
        <img [src]="student.avatar" alt="Аватар" class="avatar">
   
        <!-- Interpolation -->
        <p><strong>ФИО:</strong> {{ student.name }}</p>
        <p><strong>Группа:</strong> {{ student.group }}</p>
      </div>
    } @else {
      <p class="loading">Загрузка данных профиля...</p>
    }
   
    <div class="test-zone">
      <h4>Эксперимент с OnPush:</h4>
      <!-- Вывод обычной переменной -->
      <p>Счетчик кликов: {{ clickCount }}</p>
      <!-- Event Binding (click) -->
      <button (button) (click)="badIncrement()">Нажать (Обычная переменная)</button>
    </div>
  </div>
```
   
## Шаг 4: Анализ поведения OnPush на практике (15 минут)

   1. Подключить ```<app-student-profile />``` в app.component.ts (не забыв добавить его в массив imports).
   2. Запустить приложение. Обратить внимание, что первые 1000 мс горит надпись «Загрузка...», а после обновления Сигнала ```studentData.set(...)``` интерфейс мгновенно отображает карточку.
   3. Эксперимент: Нажать на кнопку «Нажать (Обычная переменная)».
   * Результат: Значение в консоли увеличивается, но на экране остается 0.
      * Объяснение для студентов: Так как компонент изолирован в режиме OnPush, Angular игнорирует мутации обычных полей класса. Чтобы интерфейс среагировал, это поле тоже должно стать Сигналом (этим мы займемся на Неделе 4).
   
## Шаг 5: Проверка очистки памяти в ngOnDestroy (15 минут)

   1. Чтобы сымитировать уничтожение компонента, временно обернем его в родительском шаблоне app.component.html в условие, которое можно отключить, или просто покажем в консоли, как при переходе на несуществующий роут (или при программном скрытии компонента) срабатывает ```ngOnDestroy``` и останавливается ```setInterval```.
