## Неделя 13. Реактивные формы (Reactive Forms)
------------------------------
## ЛЕКЦИЯ 13: Архитектура Реактивных форм и управление вводом в Angular 22
Продолжительность: 2 академических часа (90 минут)

## 1. Два подхода к созданию форм в Angular 22 (20 минут)
В Angular 22 существуют два разных инструмента для работы с формами: Шаблонные (Template-driven) и Реактивные (Reactive) формы.
Необходимо зафиксировать, почему в ИТ-индустрии для сложных корпоративных проектов стандартом являются именно Реактивные формы:

* Template-driven (Шаблонные): Логика описывается прямо в HTML-разметке с помощью директивы ```[(ngModel)]```. Они просты в написании, но их тяжело тестировать юнит-тестами, в них сложно реализовать динамическое изменение полей, а вся логика оказывается размазана по HTML.
* Reactive Forms (Реактивные): Модель и структура формы создаются строго в коде TypeScript. Шаблон HTML лишь «подписывается» на готовую программную модель. Они обеспечивают полный контроль над потоком данных, поддерживают реактивные подписки на изменение каждого символа и легко покрываются изолированными тестами.

## 2. Строительные блоки: ```FormControl```, ```FormGroup``` и ```FormBuilder``` (25 минут)
Архитектура реактивной формы состоит из трех основных классов:

   1. ```FormControl```: Представляет один конкретный элемент ввода (инпут, чекбокс, дропдаун). Хранит его текущее значение, статус валидности и историю изменений.
   2. ```FormGroup```: Группа из нескольких ```FormControl``` (или других групп). Она объединяет их в единый JSON-объект. Если хотя бы один инпут внутри группы невалиден, вся группа получает статус ```invalid```.
   3. ```FormBuilder```: Сервис-помощник (синтаксический сахар), который позволяет объявлять сложные вложенные группы без постоянного вызова оператора ```new FormControl()```.

## 3. Эволюция в Angular 22: Строгая типизация и Signal Forms (25 минут)
Долгое время реактивные формы Angular страдали от отсутствия типизации — метод ```form.value``` возвращал тип ```any```. Начиная со стабильных релизов последних версий и закрепившись в Angular 22, формы стали строго типизированными.

* Тип формы автоматически выводится из структуры данных: ```FormGroup<{ title: FormControl<string> }>```. Это исключает опечатки в именах полей при компиляции.
* Signal Forms API: В Angular 22 значение формы и её статус (```valid```/```invalid```) теперь можно читать напрямую как Angular Signals с помощью нативных свойств (например, ```form.valueSignal()``` или ```control.statusSignal()```). Это позволяет OnPush-компонентам мгновенно и точечно перерисовывать элементы интерфейса (например, блокировать кнопку отправки) без привлечения RxJS-подписок.

## 4. Жизненный цикл и отслеживание состояний инпута (20 минут)
Каждое поле формы в любой момент времени обладает набором булевых флагов:

* ```pristine``` / ```dirty```: Значение поля еще НЕ менялось пользователем / Пользователь изменил значение хотя бы раз.
* ```untouched``` / ```touched```: Пользователь еще НЕ заходил в поле (не было фокуса) / Пользователь кликнул на поле и ушел из него (```blur```), что является идеальным моментом для показа сообщения об ошибке.

## ПРАКТИЧЕСКОЕ ЗАНЯТИЕ 13: Создание формы добавления учебного курса / задачи
Продолжительность: 2 академических часа (90 минут)
Цель работы: Подключить инструменты реактивных форм, спроектировать строго типизированную модель формы с помощью ```FormBuilder```, связать её с HTML-разметкой и реализовать реактивный вывод текущего состояния полей на экран в режиме OnPush.

## Шаг 1: Подключение ReactiveFormsModule (15 минут)
Поскольку проект строится на Standalone-архитектуре, модуль для работы с формами импортируется точечно в тот компонент, где эта форма будет находиться.

   1. Сгенерировать компонент формы:
```bash
  ng g c components/course-form --standalone
```
   2. Открыть course-form.component.ts и импортировать ReactiveFormsModule:
```typescript
  import { Component, ChangeDetectionStrategy, inject } from '@angular/core';
  import { ReactiveFormsModule, FormBuilder } from '@angular/forms'; // Импортируем модули форм
   
  @Component({
    selector: 'app-course-form',
    standalone: true,
    imports: [ReactiveFormsModule], // Подключаем прямо сюда
    templateUrl: './course-form.component.html',
    styleUrl: './course-form.component.scss',
    changeDetection: ChangeDetectionStrategy.OnPush
  })
  export class CourseFormComponent {
    private fb = inject(FormBuilder); // Внедряем помощник через inject()
  }
```
   
## Шаг 2: Проектирование типизированной модели формы (25 минут)
Опишем структуру будущей формы создания сущности (например, ИТ-курса) в TypeScript-классе. Зададим дефолтные значения.
```typescript
  // Внутри класса CourseFormComponent

  // Создаем группу полей с помощью FormBuilder
  courseForm = this.fb.group({
    // Поле названия: строка, начальное значение пустая строка ''
    title: this.fb.control<string>('', { nonNullable: true }),
    
    // Поле объема часов: число, начальное значение 16
    hours: this.fb.control<number>(16, { nonNullable: true }),
    
    // Поле сложности курса (дропдаун)
    level: this.fb.control<string>('beginner', { nonNullable: true })
  });

  formValueSignal = toSignal(this.courseForm.valueChanges, { 
    initialValue: this.courseForm.value 
  });
  
  formStatusSignal = toSignal(this.courseForm.statusChanges, { 
    initialValue: this.courseForm.status 
  });

  // Метод обработки отправки формы
  onSubmit(): void {
    if (this.courseForm.invalid) return;

    // Благодаря типизации v22, formData имеет строгий тип, а не any!
    const formData = this.courseForm.getRawValue();
    console.log('Данные формы готовы к отправке на бэкенд через API:', formData);
    
    // Имитируем сброс формы к дедфолтным значениям
    this.courseForm.reset();
  }
```
Примечание: Опция ```{ nonNullable: true }``` гарантирует, что при вызове метода ```.reset()``` значение поля вернется к стартовому, а не превратится в ```null```.

## Шаг 3: Связывание модели с HTML-шаблоном (20 минут)
Открыть course-form.component.html и спроектировать разметку. Для связывания используются директивы ```formGroup``` и ```formControlName```:
```html
<div class="form-container">
  <h3>Добавление нового ИТ-курса в план</h3>

  <!-- На уровне тега form связываем программную модель группы -->
  <form [formGroup]="courseForm" (ngSubmit)="onSubmit()">
    
    <div class="input-field">
      <label for="title">Название дисциплины:</label>
      <!-- Связываем конкретный FormControl через строковое имя -->
      <input id="title" type="text" formControlName="title" placeholder="Например, Архитектура Angular 22">
    </div>

    <div class="input-field">
      <label for="hours">Количество академических часов:</label>
      <input id="hours" type="number" formControlName="hours">
    </div>

    <div class="input-field">
      <label for="level">Уровень сложности:</label>
      <select id="level" formControlName="level">
        <option value="beginner">Начальный (Бакалавриат)</option>
        <option value="advanced">Продвинутый (Магистратура)</option>
      </select>
    </div>

    <button type="submit" class="submit-btn">Сохранить на сервере</button>
  </form>
</div>
```

## Шаг 4: Реактивный дебаг состояния (15 минут)
Добавим в подвал шаблона блок, наглядно демонстрирующий работу Signal Forms в Angular 22. Будем выводить JSON текущих значений формы в реальном времени.
```html
<div class="debug-panel">
  <h4>Отладка состояния Reactive Form (Zoneless + OnPush):</h4>
  
  <!-- Читаем наше сконвертированное значение как Сигнал -->
  <pre>Значение полей: {{ formValueSignal() | json }}</pre>
  
  <p>Статус валидности формы: 
    <span [class.valid-text]="formStatusSignal() === 'VALID'">
      {{ formStatusSignal() }}
    </span>
  </p>
</div>
```

## Шаг 5: Тестирование ввода (15 минут)

   1. Подключить ```<app-course-form />``` в app.component.ts.
   2. Запустить dev-сервер: ```ng serve```.
   3. Начать вводить текст в инпуты. Обратить внимание на блок дебага: благодаря встроенной интеграции форм с Сигналами, OnPush-компонент моментально обновляет JSON-строку на экране при нажатии каждой клавиши, без написания ручных обработчиков событий (input).
