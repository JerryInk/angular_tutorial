## ЛЕКЦИЯ 5: Современный обмен данными между компонентами в Angular 22
Продолжительность: 2 академических часа (90 минут)

## 1. Сигнальные инпуты: Функция ```input()``` взамен ```@Input()``` (25 минут)
Долгое время для передачи данных от родительского компонента к дочернему использовался декоратор ```@Input()```. В Angular 22 этот подход признан устаревшим в рамках Signals-First архитектуры. Ему на смену пришли сигнальные инпуты (Signal Inputs), реализуемые через функцию ```input()```.

* Что это такое: Сигнальный инпут — это создаваемый Read-Only Signal внутри дочернего компонента, значение которого контролирует родитель.
* Синтаксис:
```typescript
  // Необязательный инпут со значением по умолчанию:
  title = input<string>('Заголовок по умолчанию'); 
  // Обязательный инпут (компилятор выдаст ошибку, если родитель его не передаст):
  courseId = input.required<number>();
```
* Преимущества перед старым ```@Input()```:
  1. Интеграция с ```computed()``` и ```effect()```: Поскольку инпут теперь является полноценным сигналом, вы можете мгновенно создавать производные вычисления: ```isShortCourse = computed(() => this.hours() < 16)```. Больше не нужен сложный перехватчик ```ngOnChanges()```.
  2. Строгая типизация: Шаблонизатор Angular 22 идеально проверяет типы данных, передаваемых в сигнальные инпуты, прямо на этапе компиляции.

## 2. Современная генерация событий: Функция output() (20 минут)
Для отправки действий пользователя или сигналов наверх (от дочернего компонента к родительскому) в Angular 22 используется функция ```output()``` вместо старого декоратора ```@Output()```.

* Синтаксис:
```typescript
  deleteEvent = output<number>(); // Излучает ID удаляемого элемента
```
* Как использовать: Чтобы сгенерировать событие, вызывается метод ```.emit()```:
```typescript
  this.deleteEvent.emit(this.courseId());
```
* Преимущество: Новый ```output()``` больше не завязан на тяжелые механизмы RxJS EventEmitter под капотом, что делает отправку событий в Zoneless-режиме значительно более легковесной и быстрой.

## 3. Проекция контента с помощью ```<ng-content>``` (20 минут)
Иногда компоненту нужно передать не просто строку или массив, а целый кусок HTML-разметки с кастомной версткой. Для этого используется механизм проекции контента (Content Projection).

* Внутри дочернего компонента ставится тег-заглушка: ```<ng-content />```.
* Родитель пишет разметку прямо между открывающим и закрывающим тегом дочернего компонента:
```html
<app-custom-card>
  <p>Этот текст спроецируется внутрь карточки!</p>
</app-custom-card>
```
* Мульти-проекция: С помощью атрибута ```select``` можно разносить разные куски разметки по разным местам дочернего шаблона (например, ```<ng-content select="[card-header]" />``` и ```<ng-content select="[card-body]" />```).

## 4. Практическое применение паттерна Smart/Dumb в связке с Signals (25 минут)
В Angular 22, благодаря полному отсутствию зон и повсеместному использованию ```OnPush```, эта схема гарантирует точечное и молниеносное обновление интерфейса.
```
        SMART COMPONENT (Контейнер / "Мозг" страницы)
  ┌────────────────────────────────────────────────────────────┐
  │  // Состояние (Источник правды)                            │
  │  courses = signal([...]);                                  │
  │                                                            │
  │  // Метод обновления состояния                             │
  │  deleteCourse(id) {                                        │
  │    this.courses.update(list => list.filter(c=>c.id!==id)); │
  │  }                                                         │
  └──┬─────────────────────────────────────────────────────────┘
     │                                       (Smart-компонент перехватывает событие в HTML-шаблоне:
     │                                       (onDelete)="deleteCourse($event)" и обновляет Сигнал)
     │                                                ▲
  ПОТОК ДАННЫХ ВНИЗ (Data Down)                       │
  Передача реактивного значения                       │
  через свойство: [course]="item"            ПОТОК СОБЫТИЙ ВВЕРХ (Events Up)
     │                                       Излучение легковесного события
     │                                       через функцию: onDelete = output<number>()
     │                                                │
     ▼  DUMB COMPONENT (Презентация / Карточка UI)    │
  ┌───────────────────────────────────────────────────┴────┐
  │  // Входной сигнал (Read-Only)                         │
  │  course = input.required<CourseData>();                │
  │                                                        │
  │  // Вызов действия пользователя (Клик по кнопке)       │
  │  triggerDelete() {                                     │
  │    this.onDelete.emit(this.course().id); ◄─────────────── Пользователь
  │  }                                                     │
  └────────────────────────────────────────────────────────┘
```
   1. Поток данных ВНИЗ (Data Down):
      * Smart-компонент полностью владеет центральным состоянием приложения. Данные хранятся в изменяемом Сигнале ```signal()```.
      * Внутри HTML-шаблона Smart-компонента развернут цикл ```@for```. На каждой итерации он спускает объект данных в дочерний компонент с помощью привязки свойств: ```[course]="item"```.
      * Dumb-компонент принимает эти данные с помощью современного сигнального инпута ```input.required()```. Внутри него эти данные доступны только для чтения. Любые производные вычисления (например, проверка длительности курса) делаются на лету с помощью мемоизированного ```computed()```.
   2. Поток событий ВВЕРХ (Events Up):
      * Пользователь совершает интерактивное действие внутри Dumb-компонента (например, нажимает на кнопку «Удалить из плана»).
      * Dumb-компонент перехватывает нативный клик через ```(click)``` и вызывает внутренний метод, который производит выброс события наверх: ```this.onDelete.emit(id)```. Для этого используется новая производительная функция ```output()```.
      * Smart-компонент слушает это событие в своем шаблоне через конструкцию ```(onDelete)="deleteCourse($event)"```. Он перехватывает переданный ID, вызывает метод ```.update()``` у своего исходного Сигнала и иммутабельно изменяет массив.
      * Граф зависимостей Angular моментально реагирует на изменение Сигнала, и интерфейс точечно перерисовывается. Цикл замыкается.

## ПРАКТИЧЕСКОЕ ЗАНЯТИЕ 5: Декомпозиция каталога курсов на Smart и Dumb компоненты
Продолжительность: 2 академических часа (90 минут)
Цель работы: Применить паттерн Smart/Dumb на практике, заменить старые декораторы современными функциями ```input()``` и ```output()```, реализовать передачу данных и событий между компонентами в изолированном OnPush-режиме.

## Шаг 1: Создание Глупого (Dumb) компонента карточки (20 минут)

   1. Сгенерировать новый презентационный компонент, который будет отвечать исключительно за отрисовку одной карточки курса:
```bash 
  ng g c components/course-card
```
   2. Открыть course-card.component.ts и настроить сигнальные инпуты и аутпуты в режиме OnPush:
```typescript
  import { Component, ChangeDetectionStrategy, input, computed, output } from '@angular/core';
  interface CourseData {
    id: number;
    title: string;
    hours: number;
  }
   
  @Component({
    selector: 'app-course-card',
    imports: [],
    templateUrl: './course-card.component.html',
    styleUrl: './course-card.component.scss',
    changeDetection: ChangeDetectionStrategy.OnPush // Строго OnPush
  })
  export class CourseCardComponent {
    // Принимаем обязательный объект курса через Signal Input
    course = input.required<CourseData>();
    
    // Вычисляемый сигнал прямо на базе инпута (фича v22)
    isIntensive = computed(() => this.course().hours > 40);
    
    // Событие для уведомления родителя об удалении курса
    onDelete = output<number>();
    
    // Метод обработки клика по кнопке удаления
    triggerDelete(): void {
      this.onDelete.emit(this.course().id);
    }
  }
```
   
## Шаг 2: Верстка шаблона Глупого компонента (15 минут)

   1. Открыть course-card.component.html. Обратите внимание, что к инпуту мы обращаемся как к функции ```course()```:
```html
  <div class="card-item" [class.intensive]="isIntensive()">
    <h4>{{ course().title }}</h4>
    <p>Длительность: {{ course().hours }} часов</p>
    
    @if (isIntensive()) {
      <span class="badge-warn">Интенсивный курс</span>
    }
    
    <!-- Кнопка сигнализирует наверх, но сама данные не удаляет -->
    <button (click)="triggerDelete()" class="delete-btn">Удалить из плана</button>
  </div>
```
   
## Шаг 3: Модернизация Умного (Smart) компонента (25 минут)
В роли Smart-компонента выступит созданный ранее ```CourseListComponent```. Он будет управлять общим состоянием.

   1. Открыть course-list.component.ts и переписать логику:
```typescript
  import { Component, ChangeDetectionStrategy, signal } from '@angular/core';
  import { CourseCardComponent } from '../course-card/course-card.component'; // Импортируем Dumb-компонент
   
  @Component({
    selector: 'app-course-list',
    imports: [CourseCardComponent], // Обязательно импортируем дочерний Standalone-компонент
    templateUrl: './course-list.component.html',
    styleUrl: './course-list.component.scss',
    changeDetection: ChangeDetectionStrategy.OnPush
  })
  export class CourseListComponent {
    // Центральное состояние данных (Smart-компонент имеет к нему доступ)
    courses = signal([
      { id: 1, title: 'Архитектура Angular 22', hours: 64 },
      { id: 2, title: 'Проектирование REST API', hours: 32 },
      { id: 3, title: 'Основы DevOps и CI/CD', hours: 16 }
    ]);
    
    // Метод обработки события, пришедшего снизу от карточки
    deleteCourseFromState(idToDelete: number): void {
      console.log(`Smart-компонент перехватил удаление курса с ID: ${idToDelete}`);
      // Иммутабельно обновляем состояние через .update()
      this.courses.update(currentList => currentList.filter(c => c.id !== idToDelete));
    }
  }
```
   
## Шаг 4: Связывание компонентов в шаблоне родителя (15 минут)

   1. Открыть course-list.component.html и заменить старую инлайновую верстку вызовами дочернего компонента внутри цикла ```@for```:
```html
  <div class="smart-container">
    <h3>План обучения студента (Управление компонентами)</h3>
  
    <div class="cards-list">
      @for (item of courses(); track item.id) {
        <!-- Передаем данные в инпут [course] и слушаем аутпут (onDelete) -->
        <app-course-card 
          [course]="item" 
          (onDelete)="deleteCourseFromState($event)" 
        />
      } @empty {
        <p class="empty-msg">Все курсы удалены из плана.</p>
      }
    </div>
  </div>
```
   
## Шаг 5: Тестирование сквозного потока данных (15 минут)

   1. Убедиться, что ```CourseListComponent``` подключен к ```AppComponent```.
   2. Запустить проект. Проверить в браузере: при клике на кнопку «Удалить из плана» в любой карточке, глухой компонент успешно сообщает об этом умному родителю, родитель обновляет свой Сигнал ```courses```, и интерфейс мгновенно перерисовывается.
   3. Посмотреть с помощью расширения Angular DevTools в браузере, что обновляются только измененные DOM-элементы, а изоляция OnPush работает идеально.
